Technique 02 · Constraint Mapping

Which walls are real, which are imagined

The stack is fixed (Vercel + Neon + Upstash). So we sorted every worry into a genuine limit vs. a solvable one. Only one true wall stood.

ConcernVerdictWhy
Remote / HTTP MCP serversImaginedWork fine from Vercel serverless — the agent route acts as MCP client.
Local stdio / Docker MCP serversREAL wallCan't run on serverless. Route around via a small always-on host, or provider-hosted connectors.

★ Star idea

The Capability-Gap Feedback Ledger

When an agent hits something it can't do in this environment, it never says "I can't." It says "great idea — noted for the builder." The gap is logged with its why, task context, and frequency — becoming an evidence-backed, auto-prioritized build backlog. Graceful degradation as delight. (Very BMad — it mirrors the deferred-work ledger / memlog pattern.)

agent detects gap mid-flow → logged w/ why + context + frequency → consent "outbox" she reviews → sent to builder (Bloompzah)

Implementation seed: each agent carries a capability manifest in its system prompt — what THIS env can and can't do — so it self-recognizes gaps accurately. The rule: whatever can't be done in this environment becomes part of the document.

The Crudo Discovery · a pivot

From "a BMad UI" to a "second brain"

Researching the private-repo folder surfaced context that reshaped the whole thesis.

FOLDER FINDING

Crudo already runs today

A working BMad custom agent — "GTM Product SEO Strategist" — codifying Vee's SEO/AEO IP. 20 skills, 9 workflows, an SEO rules engine, real client audits. Runs now as the CLI command /crudo.

THE MATCH

The "friend" is Vee

The persona exactly matches the fractional GTM/SEO/AEO strategist the whole private-repo vision is built around. Prior briefs even spec a "command-and-record web app" — but assumed OpenClaw. No Vercel/Neon/Upstash UI exists yet. This brainstorm's UI is the missing app layer.

RECONCILIATION

Brainstorming-first, not Crudo-first

v1 (Mary + brainstorming + canvas) = the disarming on-ramp that builds trust before the high-stakes killer app. Its output artifact becomes the input to Crudo. One funnel, two agents.

PIVOT

A general-purpose second brain

Not SEO-locked. She works on all sorts of problems (her new US/Canada client isn't even SEO). Crudo = future context, not v1. "Second brain" = selective, opt-in memory across sessions and clients — but de-prioritized, because success isn't scale.

The Synthesis · why this exists

Three intents. One aha. Ten genuine uses.

Underneath the product is a purpose. This isn't built to scale — it's an educational proof-of-concept that seeds the next thing for both of them.

INTENT 01

Meet BMad's method

Show her ChatGPT isn't the best way. Better techniques exist — you just need a proper framework.

INTENT 02

Meet the agents

What agents are, their limitations, and the limits of AI itself. AI is not magic — the honest "I can't, but noted" moments are the lesson.

INTENT 03

Prove structure wins

A structured approach beats ChatGPT on the actual outcome — a document she walks away holding.

~10
genuine uses = success

Not retention, not scale, not an 18-month moat. Just enough real usage to spark the aha, teach the lesson, and give both of them enough to work on next. A foundation, not a finish line.

The capability-gap ledger and the agent being honest about limits is the mechanism that delivers Intent 02 — the "dead ends" we worried about in Chaos are actually the pedagogy. Showing the seams teaches her AI isn't magic.
